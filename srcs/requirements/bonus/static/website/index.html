Project Overview

The Inception project requires setting up a small infrastructure using Docker containers with specific services and configurations. The project must be completed on a Virtual Machine using Docker Compose.

Mandatory Requirements

Container Setup

Mandatory Each service must run in a dedicated container with the same name as the service.

Containers must be built from either Alpine or Debian (penultimate stable version).

Example Dockerfile:

FROM alpine:3.16 RUN apk update && apk add nginx COPY nginx.conf /etc/nginx/nginx.conf EXPOSE 443 CMD ["nginx", "-g", "daemon off;"]

Required Services

  • NGINX with TLSv1.2/1.3 only
  • WordPress with php-fpm (no nginx)
  • MariaDB (no nginx)

NGINX TLS Configuration:

ssl_protocols TLSv1.2 TLSv1.3; ssl_ciphers 'TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384'; ssl_prefer_server_ciphers on;

Volumes & Network

  • Volume for WordPress database
  • Volume for WordPress files
  • Docker network for container communication

docker-compose.yml snippet:

volumes: wordpress_db: driver: local driver_opts: type: none device: /home/yourlogin/data/db o: bind networks: inception_net: driver: bridge

Security

  • No passwords in Dockerfiles
  • Environment variables mandatory
  • Use of .env file recommended
  • Docker secrets for sensitive data

.env file example:

DOMAIN_NAME=yourlogin.42.fr MYSQL_USER=wp_user MYSQL_PASSWORD_FILE=/run/secrets/db_password MYSQL_ROOT_PASSWORD_FILE=/run/secrets/db_root_password

Domain Configuration

Mandatory Domain must be login.42.frReplace 'login' with your actual login pointing to local IP.

NGINX must be the sole entry point (port 443 only).

Edit your /etc/hosts file:

127.0.0.1 yourlogin.42.fr

Database Users

Mandatory Two users required in WordPress DB.

Important Admin username must NOT contain: admin, Admin, administrator, Administrator.

Creating users:

CREATE USER 'wp_user'@'%' IDENTIFIED BY 'password'; CREATE USER 'site_manager'@'%' IDENTIFIED BY 'strongpassword'; GRANT ALL PRIVILEGES ON wordpress.* TO 'site_manager'@'%'; GRANT SELECT, INSERT, UPDATE ON wordpress.* TO 'wp_user'@'%';

Bonus Opportunities

Redis Cache

Bonus Set up Redis cache for WordPress.

Improves performance by caching database queries and sessions.

1. Add Redis container to docker-compose.yml

2. Install Redis Object Cache plugin in WordPress

3. Configure wp-config.php:

define('WP_REDIS_HOST', 'redis'); define('WP_REDIS_PORT', 6379);

FTP Server

Bonus FTP server container pointing to WordPress volume.

Allows easy file management for WordPress content.

docker-compose.yml entry:

ftp: image: stilliard/pure-ftpd volumes: - wordpress_files:/home/ftpuser ports: - "21:21" - "30000-30009:30000-30009" environment: FTP_USER_NAME: ftpuser FTP_USER_PASS: secret FTP_USER_HOME: /home/ftpuser

Static Website

Bonus Create a simple static website (non-PHP).

This page qualifies for this bonus!

Simple static site Dockerfile:

FROM nginx:alpine COPY . /usr/share/nginx/html EXPOSE 80

Adminer

Bonus Set up Adminer database management.

Provides a web interface for database administration.

docker-compose.yml entry:

adminer: image: adminer restart: always ports: - 8080:8080 networks: - inception_net

Custom Service

Bonus Add a service of your choice with justification.

Suggestions: Monitoring (Grafana), Backup (Duplicati), or CI/CD (Jenkins).

Grafana for Monitoring:

grafana: image: grafana/grafana ports: - 3000:3000 volumes: - grafana_data:/var/lib/grafana networks: - inception_net

Key Prohibitions

  • No ready-made Docker images (except Alpine/Debian)
  • No network: host or --link
  • No infinite loop commands (tail -f, sleep infinity, etc.)
  • No "latest" tag in images
  • No admin-like usernames
